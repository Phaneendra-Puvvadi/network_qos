❓ How to Use This System

Complete setup and usage guide for Linux

📋 Prerequisites

Before starting, ensure you have:

  • Linux system (Ubuntu, Kali, or similar)
  • Terminal access with sudo privileges
  • Internet connection for package installation

📦 Step 1: Install Required Packages

Open terminal and run this command:

sudo apt-get update && sudo apt-get install -y build-essential libpcap-dev iperf3 net-tools iproute2 tcpdump wireshark python3 python3-pip libjson-c-dev

Install Python packages:

pip3 install flask reportlab --break-system-packages

🔧 Step 2: Setup Network Environment

Create virtual network namespaces:

cd ~/network_qos
./setup_network.sh

You should see confirmation:

=== Network setup complete ===
Client: 10.0.1.2
Router: 10.0.1.1 / 10.0.2.1
Server: 10.0.2.2

🚀 Step 3: Start the Router

Terminal 1: Run the router program

sudo ip netns exec router ./network_qos

Expected output:

Router running. Press Ctrl+C to stop.

🌐 Step 4: Start Web Dashboard

Terminal 2: Start the web server

cd ~/network_qos/web
python3 server.py

Open browser and navigate to:

http://localhost:5000

🔍 Step 5: Start Wireshark (Optional)

Terminal 3: Monitor packets with Wireshark

sudo ip netns exec router wireshark

In Wireshark:

  • Select interface: veth-router1
  • Click "Start" to begin capture
  • Apply display filter: ip

📡 Step 6: Start iperf3 Servers

Open 6 new terminals and run ONE command in each:

Terminal 4:

sudo ip netns exec server iperf3 -s -p 5000

Terminal 5:

sudo ip netns exec server iperf3 -s -p 20000

Terminal 6:

sudo ip netns exec server iperf3 -s -p 443

Terminal 7:

sudo ip netns exec server iperf3 -s -p 80

Terminal 8:

sudo ip netns exec server iperf3 -s -p 21

Terminal 9:

sudo ip netns exec server iperf3 -s -p 5201

🚦 Step 7: Generate Traffic (Clients)

Open 6 MORE terminals and run these commands to generate traffic:

Terminal 10: High Priority Traffic (Video Call)

sudo ip netns exec client iperf3 -c 10.0.2.2 -u -b 3M -p 5000 -t 120 -l 200

Terminal 11: High Priority Traffic (VoIP)

sudo ip netns exec client iperf3 -c 10.0.2.2 -u -b 3M -p 20000 -t 120 -l 300

Terminal 12: Medium Priority Traffic (HTTPS)

sudo ip netns exec client iperf3 -c 10.0.2.2 -u -b 2M -p 443 -t 120 -l 800

Terminal 13: Medium Priority Traffic (HTTP)

sudo ip netns exec client iperf3 -c 10.0.2.2 -u -b 2M -p 80 -t 120 -l 600

Terminal 14: Low Priority Traffic (FTP)

sudo ip netns exec client iperf3 -c 10.0.2.2 -b 8M -p 21 -t 120

Terminal 15: Low Priority Traffic (Download)

sudo ip netns exec client iperf3 -c 10.0.2.2 -b 8M -p 5201 -t 120 -l 1450

👀 Step 8: Watch the Dashboard

In your web browser at http://localhost:5000, you should now see:

  • ✅ All 3 queue bars filling up (Red, Yellow, Green)
  • 📈 Congestion score rising above 0.6
  • ⚠️ Status changing to "CONGESTED" (red)
  • 📊 Real-time graph showing score fluctuations
  • 🔢 Packets/second counter increasing

⏸️ Step 9: Stop Traffic

After 2 minutes (120 seconds), all traffic will stop automatically.

Watch the dashboard:

  • Queues should drain to 0 within 3-5 seconds
  • Congestion score drops back below 0.6
  • Status returns to "NORMAL" (green)

🛑 Step 10: Cleanup

When finished, stop all programs:

  • Press Ctrl+C in Terminal 1 (router)
  • Press Ctrl+C in Terminal 2 (web server)
  • Close all iperf3 terminals

Clean up network namespaces:

cd ~/network_qos
./cleanup_network.sh

📥 Step 11: Download Report

Click the "Download Report" button in the top menu to generate a PDF report with:

  • Total packets processed
  • Current congestion status
  • Queue statistics
  • Timestamp of simulation

❗ Troubleshooting

Problem: "Permission denied" errors

Solution: Make sure you're using sudo for all network commands

Problem: Dashboard shows all zeros

Solution: Ensure router is running (Terminal 1) before generating traffic

Problem: iperf3 "Connection refused"

Solution: Start servers (Terminals 4-9) BEFORE running clients (Terminals 10-15)

Problem: Queues not draining

Solution: Wait 3-5 seconds after traffic stops. System auto-drains queues.

Again to come back to dashboard click on network Qos router
