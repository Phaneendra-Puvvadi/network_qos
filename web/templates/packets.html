🔗 TCP (Transmission Control Protocol)

Purpose: TCP provides reliable, ordered, and error-checked delivery of data between applications.

Key Features:

  • Connection-oriented (3-way handshake)
  • Guaranteed delivery with acknowledgments
  • Flow control and congestion control
  • Packet ordering and retransmission

Common Uses: Web browsing (HTTP/HTTPS), email (SMTP), file transfer (FTP), remote access (SSH)

In Our Router: TCP packets are typically classified as Low or Medium Priority depending on packet size. Large TCP packets (file downloads) get low priority, while small packets (web requests) get medium priority.

×

📡 UDP (User Datagram Protocol)

Purpose: UDP provides fast, low-latency communication without guaranteed delivery.

Key Features:

  • Connectionless protocol
  • No handshake or acknowledgments
  • Minimal overhead and latency
  • No guaranteed delivery or ordering

Common Uses: Video streaming (YouTube, Netflix), voice calls (Zoom, Skype), online gaming, DNS queries, live broadcasts

In Our Router: Small UDP packets are classified as High Priority (Real-time) because they're typically used for time-sensitive applications like video calls where delays are unacceptable.

×

🌐 HTTP/HTTPS (HyperText Transfer Protocol)

Purpose: HTTP/HTTPS is the foundation of data communication on the World Wide Web.

Key Features:

  • Request-response protocol (client-server)
  • HTTPS adds encryption (SSL/TLS)
  • Stateless protocol with cookies for sessions
  • Supports various methods (GET, POST, PUT, DELETE)

Common Uses: Web browsing, REST APIs, web applications, downloading files from websites

In Our Router: HTTP/HTTPS traffic on ports 80/443 is classified as Medium Priority (Interactive) when packets are small (requests/responses), or Low Priority (Bulk) for large downloads.

×

📞 RTP (Real-time Transport Protocol)

Purpose: RTP delivers audio and video over IP networks with timing information.

Key Features:

  • Runs on top of UDP for low latency
  • Includes timestamps and sequence numbers
  • Supports multiple media streams
  • Used with RTCP for quality monitoring

Common Uses: Video conferencing (Zoom, Teams, Google Meet), VoIP calls (WhatsApp, Skype), live streaming, IP cameras

In Our Router: RTP packets (ports 16384-32767) are automatically classified as High Priority (Real-time) because even small delays cause stuttering in video/audio calls.

×

📁 FTP (File Transfer Protocol)

Purpose: FTP is used for transferring files between computers on a network.

Key Features:

  • Uses two channels: command (port 21) and data (port 20)
  • Supports authentication and permissions
  • Can transfer large files efficiently
  • Active and passive transfer modes

Common Uses: Website file uploads, software distribution, backup systems, large file sharing

In Our Router: FTP traffic on ports 20/21 is classified as Low Priority (Bulk) because file transfers are not time-sensitive and can tolerate delays without affecting user experience.

×

🔍 ICMP (Internet Control Message Protocol)

Purpose: ICMP is used for network diagnostics and error reporting.

Key Features:

  • Network layer protocol (part of IP suite)
  • No ports (not TCP/UDP)
  • Used for error messages and queries
  • Essential for network troubleshooting

Common Uses: Ping (network reachability test), Traceroute (path discovery), error notifications (destination unreachable, time exceeded)

In Our Router: ICMP packets are typically classified as Medium Priority (Interactive) as they're important for network health monitoring but not as critical as real-time media.
